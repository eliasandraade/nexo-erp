## Design Context

### Project
**Orken** — sistema operacional empresarial modular, multi-tenant, para negócios reais brasileiros.
Não é ERP. Não é PDV simples. É a camada de controle entre o caos da operação e a decisão do dono.

Stack: React + Vite + TypeScript + Tailwind CSS + shadcn/ui + Radix UI.
Fonte: Inter. Tema: dark-first (navy/indigo). Domínio: app.orken.com.br.

### Users
**Primário:** donos de restaurantes, bares, lojas e pequenos negócios brasileiros.
Perfil: operam com planilha, WhatsApp e improviso. Não têm equipe de TI. Tomam decisão no balcão.
Contexto de uso: sob pressão, com equipe na frente, muitas abas abertas, celular na mão.
Job to be done: "quero saber se estou lucrando, sem precisar montar planilha no fim do dia."

**Secundário:** construtoras pequenas/médias e prestadores de serviço que precisam registrar despesas e movimentações sem retrabalho financeiro.

**Emoção alvo:** confiança operacional. Sensação de "está sob controle" — não ansiedade de aprender sistema novo.

### Brand Personality
**3 palavras:** moderno, inteligente, eficiente.

**Voz:** direta, operacional, sem jargão. Fala como colega experiente, não como manual de software.
Usa verbos de ação: registra, atualiza, fecha, alerta, sugere. Evita substantivos abstratos: solução, plataforma, ecossistema.

**Tom emocional:** confiante sem ser arrogante. Sabe o que faz, não precisa gritar.

### References (what to aim for)
- **Stripe:** hierarquia precisa, copy sem gordura, seções que respiram, CTA claro
- **Loom:** produto mostrado em uso, não em abstrato; demo integrada ao hero
- **Notion:** modularidade explicada com simplicidade; cada feature tem seu momento

### Anti-References (never look like)
- **SAP / Totvs / ERP corporativo:** frio, burocrático, screenshots de tabelas, jargão de TI
- **SaaS genérico americano:** hero com "The Platform for [X]", gradient genérico, social proof de logos que ninguém reconhece, copy que serve para qualquer produto
- **Startup hype:** buzzword de IA em cada parágrafo, promessa sem substância, animações que distraem

### Aesthetic Direction
**Tema:** dark mode definitivo. Navy escuro (#0f1729 range) com indigo como cor de ação e acento.
Não é "modo escuro opcional" — é a identidade visual do produto.

**Referência visual mais próxima:** Linear + Stripe dark. Limpo, sem ruído, tipografia dominante, cor usada com parcimônia.

**O que deve aparecer:**
- Tipografia grande e confiante no hero (Inter bold/extrabold)
- Espaço em branco generoso — deixa o produto respirar
- Cor indigo reservada para ação e destaque, não decoração
- UI screenshots reais do produto como evidência (não mockups genéricos)
- Microinterações sutis, nunca chamativas

**O que deve desaparecer:**
- Emojis como ícones de feature
- Cards com border genérico em grid 3x3 sem hierarquia
- "Parece familiar?" como gancho de dor — é clichê de landing SaaS
- Gradientes de cor (purple gradients, rainbow borders)
- Ícones Lucide isolados sem contexto

### Modules Architecture (landing)
**Core** (destaque principal): PDV, estoque, caixa, clientes, relatórios, caixa, usuários/RBAC.
**Orken Menu** (seção própria, secundária): restaurantes, mesas, KDS, delivery, ficha técnica/CMV.
**Orken Build** (seção própria, secundária): obras, despesas por contexto, registro inteligente.

Módulos têm identidade visual distinta dentro do sistema unificado — não são produtos separados.

### Intelligence / AI (primary differentiator)
A IA do Orken (OCR, interpretação automática, categorização sugerida) é **diferencial principal** — não feature de suporte.
Deve ter seção dedicada com fluxo concreto: usuário digita texto curto → sistema interpreta → sugere → usuário confirma.
Narrativa: "Pagou? Já está registrado." Foco no resultado, não na tecnologia.
Não usar as palavras: "inteligência artificial", "machine learning", "IA avançada". Mostrar o que acontece.

### Design Principles
1. **Operacional primeiro** — cada seção responde "o que muda na minha operação?" antes de qualquer abstração. Nunca explicar o que o sistema é; mostrar o que ele faz agora.
2. **Inteligência visível** — a automação se manifesta como resultado tangível (texto virou lançamento, venda virou estoque atualizado), não como promessa de feature.
3. **Premium acessível** — visual de produto sofisticado, linguagem de pessoa real. O design eleva; a copy aproxima.
4. **Hierarquia sem ansiedade** — múltiplos módulos sem confundir. O usuário entende o Core em 10 segundos; os módulos verticais são um "tem mais se você precisar".
5. **Evidência, não abstração** — screenshots, números reais, fluxos concretos. Nenhuma seção que só promete sem mostrar.
